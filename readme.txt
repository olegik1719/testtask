Тестовое задание.

Условие задачи.

Сделать приложение для быстрого (текстового) поиска по содержимому загруженных файлов.

Функционал:
- загрузка файла, файлы можно загружать любым способом, главное чтобы я смог воспроизвести;
- быстрый поиск по всем загруженным файлам (вхождение строки);
- логгирование (Тут интересно помотреть какие данные вы посчитаете нужными логировать)
- Rest api например(поиск, выдача куска текста где встретилась подстрока, скачивание файла)

Условия:
- Файлы могут быть большого размера
- Задание тестовое, все может хранится в памяти и жить только во время жизни приложения
- Spring, Java любой версии
- минимальная длинна строки для поиска 3 символа
- Нельзя использовать готовые framework наподобие elasticsearch для поиска, суть задания сделать это самому.



Описание конечного решения:

Программа работает в консольном режиме.

Принимает от пользователя (в консоль) команды вида:
{+|-*|?*|l|e|h} <параметр[ы]>

{действие<+число, в зависимости от команды>} <строковый параметр, в зависимости от команды>

Список возможных действий:
+ -- загрузка файла. Путь задается строковым параметром;
-<номер файла> -- скачивание файла по номеру. Путь задается строковым параметром;
?<количество> -- поиск указанного количества результатов во всех файлах. Строка поиска задается строковым параметром;
                при отсутствии указания на количество результатов выводятся все найденные строки.
l -- список файлов. При указании строки выводит все файлы, которые содержат эту строку;
e -- выход из программы;
h -- справка по программе.
